# ControlIt — הכנה לבחינת הבגרות (הגנה) 🎓

> **הנקודה הכי חשובה:** רשימת המושגים והשאלות של הבוחן בנויה על **אנדרואיד/Java**
> (Activity, Intent, Manifest, SQLite, XML, onClick). הפרויקט שלך הוא **Python + tkinter**
> — אפליקציית **מחשב (Desktop)**, לא אנדרואיד. לכן לכל מושג אנדרואידי תענה:
> *"זה מושג מאנדרואיד; הפרויקט שלי בפייתון, והמקבילה אצלי היא ___."*
> זו בדיוק התשובה שהבוחן מצפה לה ("להסביר רק אם קיים אצלכם").

---

## 📋 חלק 1 — מיפוי המושגים (אנדרואיד → הפרויקט שלך)

| מושג (אנדרואיד) | קיים אצלי? | המקבילה / ההסבר בפרויקט שלי |
|---|---|---|
| **מחלקה (class)** | ✅ כן | `LoginApp`, `CyberDashboard`, `AgentApp`, `UDPListener`, `CommandServer` — מבנים שמכילים נתונים ופעולות |
| **פעולה בונה (constructor)** | ✅ כן | `__init__` — נקראת אוטומטית כשיוצרים עצם, מאתחלת אותו |
| **ירושה (inheritance)** | ✅ כן | `class LoginApp(tk.Tk)` יורש חלון; `class UDPListener(threading.Thread)` יורש thread |
| **Button** | ✅ כן | `tk.Button` — כפתורי SIGN IN, Screenshot, Shell |
| **onClick** | ✅ המקבילה | `command=self.handle_login` בכפתור — הפונקציה שרצה בלחיצה |
| **TextView** | ⚠️ מקבילה | `tk.Label` — תווית טקסט (כותרות, "Username:") |
| **EditText** | ⚠️ מקבילה | `tk.Entry` — שדה הקלדה (שם משתמש, סיסמה) |
| **ImageView** | ⚠️ מקבילה | `tk.Label(image=...)` — הצגת צילום המסך והלוגו |
| **Layout** | ✅ המקבילה | מנהלי פריסה: `.pack()` ו-`.grid()` שמסדרים רכיבים בחלון |
| **putExtra / העברת נתונים** | ✅ המקבילה | ארגומנט לבנאי: `ControlItLauncher(username)` מעביר את השם לחלון הבא |
| **onCreate** | ⚠️ מקבילה | `__init__` + `setup_ui()` — בונים את החלון בעלייתו |
| **setContentView** | ⚠️ מקבילה | `.pack()` / `.grid()` — מציבים רכיבים בחלון |
| **findViewById** | ❌ לא צריך | בפייתון שומרים את הרכיב במשתנה ישירות: `self.login_user = tk.Entry(...)` |
| **Activity** | ❌ אנדרואיד | המקבילה: כל **חלון** (`tk.Tk` / `Toplevel`) הוא "מסך" |
| **Intent** | ❌ אנדרואיד | המקבילה: מעבר בין חלונות ע"י `self.destroy()` ופתיחת חלון חדש |
| **Manifest** | ❌ אנדרואיד | אין. נקודת הכניסה היא `main.py` |
| **XML** | ❌ אין | אין קבצי XML; הממשק נבנה כולו **בקוד פייתון** |
| **ConstraintLayout / LinearLayout / GridLayout** | ❌ אנדרואיד | המקבילה: `pack` (דומה ל-Linear), `grid` (דומה ל-Grid) |
| **Res / Raw / Drawable** | ❌ אין | אין תיקיית משאבים; הכל בקוד |
| **Animation / MediaPlayer** | ❌ אין | לא בשימוש בפרויקט |

**משפט מפתח לבחינה:**
> "הפרויקט שלי כתוב ב-Python עם ספריית tkinter ל-GUI, ולא ב-Java/Android.
> לכן מושגים כמו Activity, Intent ו-XML הם אנדרואידיים ולא קיימים אצלי —
> אבל **הרעיון** קיים: חלון = Activity, מעבר בין חלונות = Intent, ובניית
> הממשק בקוד = במקום קובץ XML."

## 📝 חלק 2 — תשובות לשאלות הבוחן

### "ספר על הפרויקט באופן כללי / מה מטרתו / מה קיים בשוק"
ControlIt היא מערכת **שליטה וניהול מחשבים מרחוק ברשת מקומית (LAN)**. מטרה: לאפשר
למנהל (למשל מורה בכיתת מחשבים) לצלם מסכים, להריץ פקודות, לשלוח הודעות ולכבות
מחשבים — מעמדה אחת. **בשוק יש** TeamViewer ו-AnyDesk, אבל הם עובדים דרך **שרת
תיווך באינטרנט**. שלי **שונה**: חיבור **ישיר ב-LAN**, בלי שרת חיצוני, בלי תלות
באינטרנט — מהיר ופרטי יותר, מתאים בדיוק לכיתה סגורה.

### "מסך רישום והזדהות — אילו נתונים נקלטים, בדיקות קלט"
נקלטים **2 נתונים**: שם משתמש וסיסמה. נבחרו כי זה המינימום לזיהוי ייחודי ולאבטחה.
**בדיקות קלט:**
- שדות ריקים → "Fill all fields".
- בהרשמה: סיסמה ≠ אישור → "Passwords don't match".
- שם משתמש תפוס → "User exists".
- סיסמה שגויה → "Invalid credentials".

**אינטראקציה בשגיאה:** חלון `messagebox` קופץ עם הודעה ברורה — המשתמש מתקן ומנסה
שוב (ניסיונות בלתי מוגבלים, כי זה לא קריטי-אבטחה ברמה שדורשת נעילה).

### "איך אתה שומר נתונים / כמה טבלאות / כפילות"
**מסד נתונים MySQL** בשם `controlit_db` עם **3 טבלאות**:
- `users` — שם משתמש, סיסמה (מגובבת SHA-256), תאריכים.
- `user_machines` — יומן כניסות (שם, IP, זמן).
- `saved_targets` — מחשבי יעד שמורים.

הסיסמה **לעולם לא נשמרת כטקסט** — רק גיבוב חד-כיווני. **כפילות נמנעת** ע"י
`UNIQUE` על שם המשתמש + בדיקת `user_exists` לפני הוספה.
*(תוספת: כשאין MySQL, החשבונות נשמרים בקובץ JSON מקומי — אותה לוגיקה.)*

### "חיפוש / עדכון / מחיקה / הוספה / הצגה"
- **הוספה:** `register` (משתמש חדש), `save_user_machine` (כניסה).
- **הצגה:** רשימת המחשבים החיה בלוח, `get_saved_computers`.
- **עדכון:** `last_login` בכל כניסה, `save_target_computer` (upsert — עדכון אם קיים).
- **חיפוש:** שאילתות `SELECT ... WHERE` לפי שם משתמש.

### "ערוצי תקשורת בהם השתמשת" ⭐ (הלב של הפרויקט!)
- **TCP פורט 5555** — פקודות, **מוצפנות** (XOR + Base64).
- **UDP פורט 5556** — גילוי אוטומטי, **broadcast** לכל הרשת.
- **TCP פורט 5001** — העברת קבצים.

זה החלק הכי חזק שלך — תדע להסביר למה TCP לפקודות (אמין) ו-UDP לגילוי (מהיר, broadcast).

### "שימוש ביצירת עצמים"
כן, מרכזי. למשל `CyberDashboard(username)` יוצר עצם של לוח הבקרה;
`UDPBroadcaster()` יוצר עצם thread ששולח שידורים. כל חלון וכל thread הם עצמים.

### "פעולות מקבילות XML מול Java — דוגמה"
"בפרויקט שלי **אין XML**, הממשק נבנה בקוד פייתון. אבל הרעיון המקביל: באנדרואיד
כותבים `android:onClick="..."` ב-XML או `setOnClickListener` ב-Java; **אצלי**
המקבילה היא `command=self.handle_login` שמוגדר ישירות בקוד כשיוצרים את הכפתור."

### "איך הנתונים מועברים בין החלונות"
דרך **ארגומנט לבנאי**. למשל אחרי התחברות:
`launcher.ControlItLauncher(username)` — שם המשתמש מועבר לחלון הבא דרך ה-`__init__`.
זו החלופה שלי ל-`Intent.putExtra` של אנדרואיד.

### "ערוצי תקשורת / SMS / מייל / קרדיטס / תפריט"
היה כנים: "הפרויקט שלי הוא כלי **רשת ל-LAN**, לא אפליקציית טלפון. אין SMS/שיחה/מייל
כי אלה רכיבי טלפון; ערוצי התקשורת שלי הם **TCP/UDP ברשת המקומית**. אין תפריט נפתח
קבוע — הניווט הוא דרך כפתורים גדולים וברורים, שמתאים לכלי שליטה מהיר."

## 🔧 חלק 3 — "איך היית משנה את הקוד כדי ש..." (שאלות נפוצות!)

### לשנות **צבע** של כפתור
ב-`login_page.py`, כפתור ה-SIGN IN:
```python
tk.Button(left, text="SIGN IN", bg="#e74c3c", fg="white", ...)
```
- `bg=` = צבע רקע הכפתור. משנים `"#e74c3c"` (אדום) למשל ל-`"#27ae60"` (ירוק).
- `fg=` = צבע הטקסט.

### לשנות **טקסט** של כפתור
משנים את `text="SIGN IN"` ל-`text="כניסה"` — זהו.

### לשנות **מיקום / גודל** של כפתור
- מיקום בפריסת `pack`: הפרמטרים `padx`, `pady` (ריווח), `side` (צד).
- בלוח המנהל הכפתורים ב-`grid`: `btn.grid(row=0, column=1)` — משנים row/column.
- גודל: `height=3, width=15`.

### לגרום למערכת להיות **מהירה יותר**
- **גילוי מהיר יותר:** ב-`agent_gui.py`, `UDPBroadcaster` משדר כל `time.sleep(3)` —
  להקטין ל-1 שנייה (גילוי מהיר יותר, אבל יותר תעבורת רשת).
- **צילום מסך מהיר יותר:** ב-`do_screenshot` לשמור כ-**JPEG** במקום PNG —
  קובץ קטן בהרבה, מועבר מהר יותר.
- **תגובה מהירה יותר:** להקטין את ה-`timeout` בחיבורים.

### שהמסך ההתחלתי **יתחיל ממסך אחר**
- ב-`main.py`, ברירת המחדל היא `LoginApp().mainloop()`. כדי להתחיל ישר ממסך
  בחירת המצב — מחליפים ל-`ControlItLauncher("User").mainloop()`.
- או ב-`login_page.py` בתחתית: `if __name__ == "__main__": app = LoginApp()` —
  מחליפים את המחלקה שמופעלת.

## 💬 חלק 4 — שאלות "פילוסופיות" (תשובות קצרות)

**בדיקות קלט — בשביל מה?** למנוע שגיאות, קלט זדוני (SQL Injection), וחוויית
משתמש טובה. אצלי: שדות ריקים, סיסמאות תואמות, שם תפוס.

**למה מחלקה נפרדת ל-DB (my_connector)?** כדי שכל הגישה למסד תהיה במקום אחד —
אם אחליף מסד, אגע רק בקובץ אחד. הפרדת שכבות.

**קובץ טקסט מול מסד נתונים?** מסד (MySQL) טוב לנתונים מובנים, שאילתות, מניעת
כפילות. קובץ (JSON אצלי, כשאין שרת) פשוט ומקומי. בחרתי במסד כי רציתי ריכוז נתונים.

**העברת נתונים בין חלונות — בעיה אפשרית?** אם מעבירים יותר מדי נתונים זה מסבך;
חלופה: לשמור במסד ולשלוף בחלון הבא. אצלי מעבירים רק את שם המשתמש — מינימלי.

**יצירת עצמים — צריך?** כן — מאפשר לארגן קוד למבנים הגיוניים (חלון, thread, חיבור),
לעשות שימוש חוזר, ולהפריד אחריות.
